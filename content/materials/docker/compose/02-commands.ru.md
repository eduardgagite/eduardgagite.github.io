Docker Compose имеет свой набор команд, зеркально отражающий обычный Docker CLI, но работающий сразу со всем стеком сервисов.

## Запуск (up)

```bash
docker compose up -d
```
Эта команда делает магию:
1. Создает сеть (если нет).
2. Создает тома (если нет).
3. Скачивает образы (pull).
4. Запускает контейнеры.

Если вы изменили конфиг `docker-compose.yml` (например, поменяли порт), просто выполните `up` еще раз. Compose увидит изменения и пересоздаст только измененные контейнеры.

## Остановка и удаление (down)

```bash
docker compose down
```
Это обратная операция:
1. Останавливает контейнеры.
2. Удаляет контейнеры.
3. Удаляет созданную сеть.

**Важно:** Тома (Volumes) с данными **не удаляются** по умолчанию (чтобы вы случайно не стерли базу).
Если хотите удалить всё-всё, включая тома:
```bash
docker compose down -v
```

## Логи (logs)

Смотреть логи сразу всех сервисов в одном потоке (очень удобно для дебага взаимодействия):
```bash
docker compose logs -f
```
Или конкретного сервиса:
```bash
docker compose logs -f wordpress
```

## Статус (ps)

Посмотреть список контейнеров, запущенных *именно этим* docker-compose файлом:
```bash
docker compose ps
```

## Пересборка (build)

Если вы используете `build: .` в файле (собираете образ из кода), и код изменился:
```bash
docker compose up -d --build
```
Флаг `--build` принуждает Compose пересобрать образы перед запуском.

## Итого

- `up -d` — поднять всё.
- `down` — всё убрать.
- `logs -f` — смотреть, что происходит.
- `ps` — проверить статус.
