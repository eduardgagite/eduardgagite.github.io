Когда вы экспериментируете с Docker, ваша система быстро засоряется десятками остановленных контейнеров и неиспользуемых образов. Разберем, как поддерживать порядок.

## Имена контейнеров

По умолчанию Docker дает контейнерам случайные имена: `heuristic_perlman`, `focused_bose`. С ними неудобно работать.
Всегда задавайте имя явно флагом `--name`:

```bash
docker run -d --name redis-main redis
```
Теперь вы можете обращаться к нему по имени: `docker stop redis-main`.
**Ограничение:** Имя должно быть уникальным. Вы не можете запустить два контейнера с именем `redis-main` одновременно.

## Массовая очистка (Docker Prune)

Это самая приятная команда. Она удаляет **всё**, что не используется прямо сейчас:
- Остановленные контейнеры.
- Неиспользуемые сети.
- "Повисшие" образы (dangling images — те, у которых нет имени, обычно остатки от сборки).

```bash
docker system prune
```
Docker спросит подтверждение `Are you sure? [y/N]`.
Если хотите удалить вообще всё, включая неиспользуемые образы (даже именованные), добавьте флаг `-a`:
```bash
docker system prune -a
```
⚠️ **Будьте осторожны:** это удалит все образы, которые сейчас не запущены в контейнерах. При следующем запуске Docker придется качать их заново из интернета.

## Удаление конкретных ресурсов

Если `prune` для вас слишком радикален, можно удалять точечно:

Удалить **все** остановленные контейнеры (одной командой):
```bash
docker container prune
```

Удалить контейнер принудительно (даже если работает):
```bash
docker rm -f my-container
```

Удалить образ:
```bash
docker rmi nginx
```

## Итого

- Давайте контейнерам понятные имена (`--name`).
- Регулярно делайте `docker system prune`, чтобы освобождать место на диске.
- Не бойтесь удалять контейнеры — в этом суть Docker. Если данные важны, они должны лежать в Volume, а не в контейнере.
