Контейнер не живет вечно. У него есть четкий жизненный цикл, и для каждого этапа есть своя команда. Давайте пройдем этот путь на примере контейнера **nginx**.

## 1. Создание и запуск (docker run)

Эта команда делает всё сразу: скачивает образ (если нет), создает контейнер и запускает его.

```
docker run -d --name my-web nginx
```
- **-d**: detached mode (в фоне).
- **--name my-web**: даем контейнеру удобное имя (иначе Docker придумает смешное имя вроде **admiring_turing**).

Теперь контейнер в статусе **Up** (работает). Проверьте:
```
docker ps
```

## 2. Остановка (docker stop)

Если сервис больше не нужен, мы его останавливаем. Это штатное выключение: Docker посылает сигнал завершения, и приложение корректно закрывается (сохраняет данные, закрывает соединения).

```
docker stop my-web
```
Теперь контейнер в статусе **Exited**. Он не потребляет CPU и RAM, но всё еще занимает место на диске (логи, измененные файлы внутри).
Проверьте:
```
docker ps -a
```

## 3. Повторный запуск (docker start)

Остановленный контейнер можно "разбудить". Все данные внутри него, которые были изменены до остановки, сохранятся.

```
docker start my-web
```
Обратите внимание: **run** создает **новый** контейнер, а **start** запускает **существующий**.

## 4. Удаление (docker rm)

Если контейнер больше не нужен совсем (например, вы хотите создать новый с другими настройками), его нужно удалить.
**Важно:** Нельзя удалить работающий контейнер. Сначала **stop**, потом **rm**.

```
docker stop my-web
docker rm my-web
```

Если очень хочется удалить работающий контейнер сразу, можно использовать флаг **-f** (force), но это "грубое" убийство:
```
docker rm -f my-web
```

## Итоговая схема

1. **docker run** -> Создал + Запустил.
2. **docker stop** -> Остановил (процесс умер, контейнер остался).
3. **docker start** -> Снова запустил тот же контейнер.
4. **docker rm** -> Удалил мусор (контейнер исчез навсегда).
