Юнит-тесты проверяют конкретные случаи, которые вы придумали сами. Но что насчёт случаев, о которых вы не подумали? **Фаз-тестирование** (fuzz testing) — это когда Go сам генерирует случайные входные данные и ищет входы, на которых ваш код ломается.

Фаз-тесты встроены в Go начиная с версии **1.18**. Не нужны сторонние библиотеки.

## Первый фаз-тест

Представим, что у вас есть функция разворота строки:

```go
// reverse.go
package str

func Reverse(s string) string {
    runes := []rune(s)
    for i, j := 0, len(runes)-1; i < j; i, j = i+1, j-1 {
        runes[i], runes[j] = runes[j], runes[i]
    }
    return string(runes)
}
```

Юнит-тест проверит пару кейсов, а фаз-тест найдёт то, о чём вы не думали:

```go
// reverse_test.go
package str

import (
    "testing"
    "unicode/utf8"
)

func FuzzReverse(f *testing.F) {
    // Seed-корпус — начальные примеры для генератора
    f.Add("hello")
    f.Add("мир")
    f.Add("")
    f.Add("12345")

    f.Fuzz(func(t *testing.T, s string) {
        reversed := Reverse(s)

        // Свойство 1: длина не должна измениться
        if len(reversed) != len(s) {
            t.Errorf("длина изменилась: %d → %d", len(s), len(reversed))
        }

        // Свойство 2: двойной разворот = оригинал
        doubleReversed := Reverse(reversed)
        if doubleReversed != s {
            t.Errorf("двойной разворот не совпал: %q → %q → %q", s, reversed, doubleReversed)
        }

        // Свойство 3: результат — валидный UTF-8
        if !utf8.ValidString(reversed) {
            t.Errorf("невалидный UTF-8: %q", reversed)
        }
    })
}
```

## Правила фаз-тестов

1. Функция начинается с **Fuzz** (не Test).
2. Принимает **\*testing.F** (не \*testing.T).
3. **f.Add(...)** — добавляет начальные примеры (seed corpus). Типы аргументов должны совпадать с аргументами в **f.Fuzz**.
4. **f.Fuzz(func(t \*testing.T, ...))** — функция, которая вызывается с рандомными данными.

Поддерживаемые типы аргументов: **string**, **[]byte**, **int**, **int8/16/32/64**, **uint** и аналоги, **float32/64**, **bool**, **rune**.

## Запуск

```bash
# Обычный запуск (проверяет только seed-корпус)
go test -run FuzzReverse

# Запуск фаззинга (генерирует случайные данные)
go test -fuzz FuzzReverse

# Ограничить время фаззинга
go test -fuzz FuzzReverse -fuzztime 30s

# Ограничить количество итераций
go test -fuzz FuzzReverse -fuzztime 10000x
```

При обычном **go test** фаз-тесты работают как юнит-тесты — проверяют только seed-корпус. Чтобы запустить фаззинг, нужен флаг **-fuzz**.

## Что происходит при найденном баге

Когда фаззер находит вход, на котором тест падает, он:

1. Останавливается и показывает ошибку.
2. Сохраняет проблемный вход в файл **testdata/fuzz/FuzzReverse/**.
3. При следующем запуске **go test** этот кейс проверяется автоматически.

```text
--- FAIL: FuzzReverse (0.05s)
    --- FAIL: FuzzReverse/abc123 (0.00s)
        reverse_test.go:25: невалидный UTF-8: "\xc0\xa0"
```

Файл в **testdata/** — это регрессионный тест. Он останется навсегда и защитит от повторного появления бага.

## Что тестировать фаззингом

Фаззинг лучше всего подходит для:

- **Парсеров** — JSON, XML, YAML, URL, пользовательский формат.
- **Кодирования/декодирования** — проверка что encode → decode = оригинал.
- **Валидации** — убедиться, что валидатор не паникует на любом входе.
- **Сериализации** — marshal → unmarshal = оригинал.

```go
func FuzzParseJSON(f *testing.F) {
    f.Add([]byte(`{"name": "test"}`))
    f.Add([]byte(`[]`))
    f.Add([]byte(`null`))

    f.Fuzz(func(t *testing.T, data []byte) {
        var result any
        // Парсер не должен паниковать на любом входе
        _ = json.Unmarshal(data, &result)
    })
}
```

## Итого

Фаз-тестирование находит баги, о которых вы не думали, генерируя случайные входные данные. Используйте **f.Add()** для seed-корпуса, **f.Fuzz()** для тест-функции, **-fuzz** для запуска. Найденные баги автоматически сохраняются в **testdata/** как регрессионные тесты. Лучше всего подходит для парсеров, кодеков и валидаторов.
