В программировании переменная — это именованная ячейка в памяти компьютера, где мы храним данные. Чтобы компьютер понимал, как работать с этими данными (как с числами, текстом или чем-то еще), каждой переменной назначается **тип**.

Go — язык со строгой статической типизацией. Это значит:
1. **Строгая**: Вы не можете случайно сложить число 5 и слово "яблок". Компьютер не будет угадывать, что вы имели в виду, он выдаст ошибку еще до запуска программы.
2. **Статическая**: Тип переменной определяется один раз и навсегда. Если вы создали переменную для хранения возраста (число), вы не сможете завтра записать туда имя (строку).

Такой подход избавляет от множества глупых ошибок, которые часто возникают в языках вроде JavaScript или Python, где типы могут меняться на лету.

## Объявление переменных

Как сказать компьютеру: "Запомни это значение"? В Go есть два способа.

### 1. Полная форма (var)

Используем ключевое слово **var** (от variable — переменная). Этот способ максимально понятный и официальный.

```go
var username string = "admin"
var age int = 25
```

Иногда мы не знаем значение сразу, но хотим подготовить место. Тогда тип писать обязательно:

```go
var totalScore int
```

### 2. Краткая форма (:=)

Программисты ленивы и не любят писать лишнего. Если мы сразу присваиваем значение, Go сам может догадаться о типе.

Внутри функций (например, в **main**) можно использовать оператор **:=**.

```go
func main() {
    count := 10
    name := "Gopher"
    isActive := true
}
```

**Важное отличие**:
- **var** работает везде (и внутри, и снаружи функций).
- **:=** работает **только внутри функций**.

Если вы попытаетесь создать уже существующую переменную через **:=**, будет ошибка. Этот оператор именно для **создания новых**.

## Нулевые значения (Zero Values)

В некоторых языках (например, C++), если создать переменную и не дать ей значение, внутри будет "мусор" — случайные данные из памяти.

В Go всё безопасно. Если вы не дали значение, Go автоматически запишет туда "пустое" значение по умолчанию:

- Для чисел (**int**, **float**) — это **0**.
- Для логики (**bool**) — это **false**.
- Для строк (**string**) — это **""** (пустая строка).

```go
var price int
fmt.Println(price)
```

Это очень удобно: вы всегда знаете, с чего начинается жизнь переменной.

## Базовые типы данных

С какими данными мы обычно работаем?

### 1. Строки (string)
Любой текст. В Go строки всегда в двойных кавычках **"Hello"**. Они поддерживают любые символы, включая смайлики и кириллицу (UTF-8).

### 2. Целые числа (int)
Числа без дроби: 1, 10, -500, 42.
Тип **int** — самый стандартный. Он занимает столько памяти, сколько удобно вашему процессору (обычно 64 бита).

Иногда нужны числа конкретного размера:
- **int64**: Очень большие числа (например, ID в базе данных или время в миллисекундах).
- **uint**: Только положительные числа (Unsigned Integer).

### 3. Дробные числа (float64)
Числа с плавающей точкой: 3.14, 0.001, -99.9.
В 99% случаев используйте **float64** — это стандарт точности.

### 4. Логический тип (bool)
Только два значения: **true** (правда) или **false** (ложь). Используется в условиях: "Если пользователь админ (true), покажи кнопку".

## Преобразование типов

Представьте, что у вас есть две переменные:
- **a** типа **int** (обычное число)
- **b** типа **int64** (большое число)

В Go их **нельзя** просто сложить. Для компьютера это разные типы данных, даже если оба — числа.

```go
var a int = 10
var b int64 = 20

c := int64(a) + b
```

Это называется **явное приведение типов**. Вы говорите компилятору: "Я знаю, что делаю, преврати **a** в **int64**".

Чтобы сделать из числа строку (например, число 42 в текст "42"), простого приведения недостаточно. Нужны специальные функции из пакета **strconv** (String Converter), но об этом позже.

## Константы

Иногда нам нужны значения, которые **никогда** не меняются. Например, число Пи или название приложения. Для этого есть **const**.

```go
const Pi = 3.14
const AppName = "MyService"
```

### Зачем использовать константы?

1. **Читаемость**. Число **86400** в коде ничего не говорит новичку. А **const SecondsInDay = 86400** сразу объясняет: это количество секунд в сутках.
2. **Защита**. Вы не сможете случайно написать **Pi = 5** где-то в середине кода. Компилятор ударит по рукам.
3. **Оптимизация**. Компьютер работает с константами быстрее, так как знает их значения заранее.

### Продвинутая фишка: Нетипизированные константы

Константы в Go могут быть "умными". Пока вы не использовали константу, она не имеет жесткого типа.

```go
const Timeout = 5

var x int = Timeout
var y float64 = Timeout
```

Если бы **Timeout** была обычной переменной (**var Timeout int = 5**), вторая строка вызвала бы ошибку. Константы позволяют писать более гибкий код.

## Итог

1. Используйте **:=** внутри функций — это быстро и удобно.
2. Используйте **var**, когда объявляете переменные на уровне пакета или хотите оставить значение "по умолчанию".
3. Типы строгие: нельзя сложить **int** и **float** без явного преобразования.
4. Константы **const** делают код понятнее и безопаснее.

## Практика

1. Объявите переменные для имени (string), возраста (int) и зарплаты (float64) сотрудника. Выведите их в формате: `Имя: Alice, Возраст: 30, Зарплата: 75000.50`.
2. Напишите программу, которая объявляет две переменные типа `int` и `float64` и выводит их сумму. Подсказка: понадобится преобразование типов.
3. Создайте константы для дней недели (от 1 до 7) и напишите функцию, которая по номеру дня возвращает его название.
