Любая программа на Go состоит из пакетов (packages). Это способ разложить код по папкам, чтобы не сваливать всё в одну кучу.

## Как устроен пакет

Пакет — это просто папка с файлами `.go`.
В начале каждого файла **обязательно** должна быть строка `package имя_пакета`.

```go
// Файл: math/utils.go
package math

func Add(a, b int) int {
    return a + b
}
```

Все файлы в одной папке должны иметь **одинаковое имя пакета**. Нельзя в одной папке смешивать `package main` и `package math`.

## Экспорт (Публичное vs Приватное)

В Go нет ключевых слов `public`, `private` или `protected`. Вместо этого используется **регистр первой буквы**.

### С большой буквы — Public (Экспортируемое)

Если имя функции, структуры или переменной начинается с **Заглавной** буквы, она видна другим пакетам.

```go
package mylib

// ExportedFunc видна всем, кто импортирует mylib
func ExportedFunc() { ... }

type User struct { // Видна всем
    Name string    // Видно всем
    age  int       // Скрыто (доступно только внутри пакета mylib)
}
```

### С маленькой буквы — Private (Скрытое)

Если имя начинается со **строчной** буквы, оно доступно **только внутри этого пакета**. Снаружи его не видно.

```go
package mylib

// helperFunc нельзя вызвать из main
func helperFunc() { ... }
```

## Импорт пакетов

Чтобы использовать код из другого пакета, его нужно импортировать.

```go
package main

import (
    "fmt"                  // Стандартная библиотека
    "myproject/math"       // Наш локальный пакет
)

func main() {
    res := math.Add(10, 20) // Функция Add была с большой буквы, поэтому мы её видим
    fmt.Println(res)
}
```

## Пакет main

Пакет с именем `main` — особенный. Он говорит компилятору: "Это не библиотека, это программа, которую можно запустить". В этом пакете обязательно должна быть функция `main()`.

## Итог

1. **Папка = Пакет**. Все файлы в папке принадлежат одному пакету.
2. **Заглавная буква** = Доступно всем (Public).
3. **Строчная буква** = Доступно только своим (Private).
4. Импортируем через `import "путь/к/пакету"`.
