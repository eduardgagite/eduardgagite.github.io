В Go любая программа — это проект (модуль). Даже если вы пишете простой скрипт из 10 строк, правильно оформить его как модуль. Это позволяет фиксировать версии библиотек и делает проект переносимым.

Напишем классическую программу «Hello, World!», разберём каждую строчку и научимся запускать и собирать код.

## Создание модуля

Создайте новую папку для проекта и перейдите в неё:

```
mkdir hello-go
cd hello-go
```

Теперь инициализируем модуль. Имя модуля обычно совпадает с URL репозитория (например, **github.com/user/project**), но для локальных экспериментов можно написать что угодно.

```
go mod init example/hello
```

Эта команда создаст файл **go.mod** — главный конфигурационный файл проекта. Внутри он выглядит так:

```
module example/hello

go 1.22
```

Всего две строки: имя модуля и версия Go. Позже, когда вы начнете подключать библиотеки, здесь появится секция **require**.

## Написание кода

Создайте файл **main.go** и добавьте в него следующий код:

```
package main

import "fmt"

func main() {
    fmt.Println("Hello, Go!")
}
```

### Разбор каждой строки

**package main**
Первая строка любого файла указывает, к какому пакету он относится. Пакет **main** — особенный. Именно он говорит компилятору: "Это исполняемая программа, а не библиотека". Без **package main** Go не будет знать, что нужно создать бинарный файл.

**import "fmt"**
Подключаем стандартный пакет **fmt** (сокращение от "format"). Он содержит функции для ввода/вывода текста. **Println**, **Printf**, **Sprintf** — все они живут здесь.

Если вы импортируете пакет, но не используете его, **Go не скомпилируется**. Это может раздражать поначалу, но это защищает от мусорных импортов в проекте.

**func main()**
Главная функция. Точка входа в программу. Выполнение всегда начинается с неё. Она не принимает аргументов и ничего не возвращает (аргументы командной строки читаются через пакет **os**).

**fmt.Println("Hello, Go!")**
Вызываем функцию **Println** из пакета **fmt**. Она печатает текст и добавляет перенос строки в конце.

## Запуск программы

### Быстрый запуск (go run)

Для разработки и тестирования используйте **go run**. Эта команда компилирует программу во временную папку, запускает её и после завершения удаляет бинарник.

```
go run main.go
```

Вы увидите:
```
Hello, Go!
```

Можно запустить сразу все **.go** файлы в папке:

```
go run .
```

### Компиляция (go build)

Когда вы хотите получить готовый исполняемый файл, который можно скопировать на сервер:

```
go build
```

В текущей папке появится бинарник **hello** (или **hello.exe** на Windows). Имя берется из имени модуля.

```
./hello
```

Этот файл **полностью самодостаточен**. В него встроены все зависимости. На целевом сервере даже не нужен установленный Go.

### Установка (go install)

Если вы пишете утилиту, которую хотите использовать из любой папки:

```
go install
```

Бинарник скопируется в **~/go/bin/**. Если эта папка в вашем PATH, утилита станет доступна как обычная команда.

## Итог

Мы прошли полный цикл создания простой программы:
1. **go mod init** — создали основу проекта.
2. **package main** + **func main()** — написали точку входа.
3. **go run** — быстро запустили для проверки.
4. **go build** — собрали готовый бинарник для деплоя.
5. **go install** — установили утилиту глобально.
