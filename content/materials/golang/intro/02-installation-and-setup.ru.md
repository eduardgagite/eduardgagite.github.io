Чтобы начать писать на Go, нужно установить компилятор и настроить рабочее место. Процесс максимально простой: Go поставляется как один пакет, который содержит всё необходимое — компилятор, форматтер, тестовый фреймворк, инструменты документации и управления зависимостями.

В отличие от многих языков, здесь не нужно ставить отдельные менеджеры пакетов, сборщики или среды выполнения. Всё работает «из коробки».

## Установка Go

### macOS

Самый удобный способ — через Homebrew:

```bash
brew install go
```

Или скачайте `.pkg` установщик с [go.dev/dl](https://go.dev/dl/).

### Linux (Ubuntu/Debian)

```bash
# Скачиваем архив (замените версию на актуальную)
wget https://go.dev/dl/go1.22.0.linux-amd64.tar.gz

# Распаковываем в /usr/local
sudo tar -C /usr/local -xzf go1.22.0.linux-amd64.tar.gz

# Добавляем путь в PATH (в ~/.bashrc или ~/.zshrc)
export PATH=$PATH:/usr/local/go/bin
```

### Windows

Скачайте `.msi` установщик с [go.dev/dl](https://go.dev/dl/) и запустите его. Установщик сам добавит Go в PATH.

### Проверка установки

После установки откройте **новый** терминал и выполните:

```bash
go version
```

Если вы видите что-то вроде `go version go1.22.0 darwin/arm64`, значит всё работает.

Если терминал говорит "command not found", скорее всего нужно перезапустить терминал или добавить Go в переменную PATH.

## Настройка окружения

### GOPATH (история и современность)

Раньше в Go была обязательна сложная структура папок (GOPATH), где весь код должен был лежать в строго определённом месте (`~/go/src/github.com/user/project`). Это было неудобно и путало новичков.

С версии Go 1.11 появились **Go Modules**, и GOPATH потерял своё значение. Теперь вы создаете проекты **в любой папке**.

GOPATH по-прежнему существует (по умолчанию `~/go`), но используется только для хранения скачанных библиотек и установленных утилит. Вам не нужно его менять.

### Проверка окружения

Чтобы увидеть все переменные окружения Go:

```bash
go env
```

Самые важные:
- **GOROOT**: Где установлен Go (обычно `/usr/local/go`).
- **GOPATH**: Где хранятся скачанные библиотеки (`~/go`).
- **GOBIN**: Где лежат установленные утилиты (`~/go/bin`).

## Выбор редактора кода

Go отлично поддерживается большинством редакторов. Вот два самых популярных варианта:

### VS Code (бесплатный)

Лёгкий и быстрый. Для работы с Go установите официальное расширение **Go** (от Go Team at Google).

При первом открытии `.go` файла VS Code предложит установить дополнительные утилиты. Соглашайтесь на все — это даст вам:
- **Автодополнение** (gopls — Language Server).
- **Подсветку ошибок** в реальном времени.
- **Автоформатирование** при сохранении файла.
- **Отладку** с breakpoints (dlv — debugger).
- **Навигацию по коду** (переход к определению функции).

### GoLand (платный)

IDE от JetBrains. Самый мощный инструмент для Go с глубоким рефакторингом, анализом кода и удобной отладкой. Если вы привыкли к IntelliJ IDEA или PyCharm, здесь будете чувствовать себя как дома.

Есть бесплатная версия для студентов и open-source проектов.

## Полезные команды Go

Утилита `go` — ваш швейцарский нож. Вот команды, которые пригодятся с первого дня:

```bash
# Проверить версию Go
go version

# Посмотреть документацию по пакету
go doc fmt
go doc fmt.Println

# Отформатировать код (автоматически исправит отступы)
gofmt -w main.go

# Проверить код на частые ошибки (статический анализ)
go vet ./...

# Скачать и установить утилиту
go install golang.org/x/tools/gopls@latest
```

## Итог

1. Скачайте Go с [go.dev/dl](https://go.dev/dl/) или через пакетный менеджер.
2. Проверьте установку: `go version`.
3. Установите расширение Go для вашего редактора (VS Code или GoLand).
4. Забудьте про GOPATH — современный Go использует модули, и проекты можно создавать в любой папке.
