Отдельные модули **RedisJSON**, **RedisSearch** и **RedisGraph** превращают Redis в гораздо более универсальный инструмент: можно хранить и индексировать JSON‑документы, делать полнотекстовый поиск и строить графовые модели прямо поверх Redis.

Их объединяет общий принцип: сложные структуры и запросы живут ближе к памяти, а приложение общается с ними через знакомый протокол Redis.

## RedisJSON — работа с JSON-объектами

**RedisJSON** добавляет полноценное хранение JSON‑документов с возможностью изменять отдельные поля без перезаписи всей строки.

Пример записи профиля пользователя:

```
JSON.SET user:42 $ '{"name":"Alice","age":30,"tags":["dev","redis"]}'
```

Чтение всего объекта:

```
JSON.GET user:42
```

Изменение одного поля:

```
JSON.SET user:42 $.age 31
```

Получение только нужного фрагмента:

```
JSON.GET user:42 $.tags
```

Такой подход удобен, когда:

- данные естественно выглядят как JSON;
- часто нужно менять отдельные поля;
- важно не гонять лишние байты между приложением и Redis.

## RedisSearch — индексы и поиск

**RedisSearch** (часто под именем RediSearch) добавляет полнотекстовый поиск, индексы и сложные запросы по данным в Redis.

Пример создания индекса по JSON‑документам профилей:

```
FT.CREATE idx:users ON JSON PREFIX 1 user: SCHEMA $.name AS name TEXT $.age AS age NUMERIC
```

Добавление данных уже рассмотрено в RedisJSON. Теперь можно искать:

```
FT.SEARCH idx:users "@name:Alice"
FT.SEARCH idx:users "@age:[25 35]"
```

RedisSearch сам ведёт индекс и позволяет:

- делать фильтрацию по полям;
- добавлять сортировку и пагинацию;
- строить сложные запросы над данными, уже лежащими в Redis.

Это удобно, когда нужно быстрый поиск по профилям, товарам или логам без отдельной поисковой системы.

## RedisGraph — графовые данные

**RedisGraph** добавляет графовую модель: вершины, рёбра и запросы в стиле Cypher.

Пример создания простого графа:

```
GRAPH.QUERY social "
  CREATE (:User {id: 1, name: 'Alice'})-[:FOLLOWS]->(:User {id: 2, name: 'Bob'})
"
```

Запрос друзей:

```
GRAPH.QUERY social "
  MATCH (u:User {id: 1})-[:FOLLOWS]->(f:User)
  RETURN f.name
"
```

Такой подход подходит для:

- социальных графов;
- рекомендательных систем;
- задач, где важны связи и пути между сущностями.

## Итог

Модули **RedisJSON**, **RedisSearch** и **RedisGraph** расширяют Redis за пределы классического «ключ–значение»: позволяют работать с JSON‑документами, делать быстрый поиск и строить графовые запросы прямо в памяти.

Если проекту нужны такие возможности, а Redis уже используется как базовая инфраструктура, эти модули позволяют обойтись без отдельных специализированных систем, сохранив при этом высокую скорость.
