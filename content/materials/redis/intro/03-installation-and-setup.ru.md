Установка Redis на Linux выполняется через стандартные системные пакеты. Ниже приведён простой рабочий набор команд, который подходит для Ubuntu, Debian, Linux Mint и других систем на базе APT.

## Установка

```
sudo apt update
sudo apt install redis-server
```

После установки сервер запускается автоматически.

## Проверка работы

Проверяем статус службы:

```
systemctl status redis-server
```

Если сервер работает, можно подключиться:

```
redis-cli
```

И отправить тестовую команду:

```
PING
```

Ожидаемый ответ:

```
PONG
```

Это означает, что всё установлено корректно.

## Управление сервером

```
sudo systemctl start redis-server
sudo systemctl stop redis-server
sudo systemctl restart redis-server
```

## Конфигурация

Основной конфигурационный файл расположен здесь:

```
/etc/redis/redis.conf
```

После изменений требуется перезапуск.

## Минимальная проверка сохранения

```
redis-cli set test "hello"
redis-cli get test
```

Если значение сохраняется после перезапуска, Redis корректно работает с диском.

## Запуск через Docker

Если не хочется устанавливать Redis в систему напрямую, его можно запустить в контейнере Docker. Это удобный способ получить изолированный экземпляр сервера.

### Запуск контейнера

```
docker run -d --name redis \
  -p 6379:6379 \
  redis:latest
```

После запуска Redis будет доступен на порту 6379, как и при обычной установке.

### Проверка работы

```
docker exec -it redis redis-cli ping
```

Ожидаемый ответ:

```
PONG
```

### Остановка и удаление контейнера

```
docker stop redis
docker rm redis
```

Этот способ подходит, если нужно быстро развернуть сервер без изменения системы или когда проект полностью работает в контейнерах.
